# МИС МКБ — Руководство по запуску

Медицинская информационная система для ведения пациентов с мочекаменной болезнью (МКБ).  
Стек: **Python 3 / FastAPI** (бэкенд) + **HTML/CSS/JavaScript** (фронтенд).

## 1. Требования

| Инструмент | Версия | Зачем |
|---|---|---|
| Python | 3.10 или новее | Запуск сервера |

## 3. Запуск бэкенда

### 3.1 Установка зависимостей

Откройте терминал в корне проекта и выполните:

```bash
pip install -r server/requirements.txt
```

Установятся два пакета:
- **fastapi** — веб-фреймворк для построения REST API
- **uvicorn** — ASGI-сервер, который запускает FastAPI-приложение

### 3.2 Запуск сервера

```bash
python -m uvicorn server.main:app --reload
```

Флаг `--reload` включает автоматическую перезагрузку при изменении кода (удобно при разработке).

**Ожидаемый вывод в терминале:**
```
INFO:     Will watch for changes in these directories: ['...\mkb-mis']
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
INFO:     Started reloader process [...]
INFO:     Started server process [...]
INFO:     Application startup complete.
```

Сервер запущен по адресу: **http://127.0.0.1:8000**

### 3.3 Проверка работоспособности

Откройте в браузере: http://127.0.0.1:8000

Должен прийти ответ:
```json
{"message": "MKB MIS server is running"}
```

Интерактивная документация API (Swagger UI): http://127.0.0.1:8000/docs

## 4. Запуск фронтенда

Фронтенд — это набор статических файлов. Открыть его можно двумя способами:

### Способ А: Простой HTTP-сервер (рекомендуется)

В **новом** окне терминала (не закрывая сервер из шага 3):

```bash
python -m http.server 8001 --directory client_interface
```

Затем откройте в браузере: **http://127.0.0.1:8001**

### Способ Б: Прямо из проводника

Просто дважды кликните на файл `client_interface/index.html` — он откроется в браузере.

> **Примечание:** Способ Б работает, но некоторые браузеры могут блокировать fetch-запросы
> при открытии файла через `file://`. Если после входа страница пустая — используйте Способ А.

## 5. Тестовые учётные данные

Данные прописаны прямо в коде (`server/main.py`) и доступны для входа сразу после запуска.

| Роль | Логин | Пароль | Что доступно |
|---|---|---|---|
| Врач | `doctor` | `1234` | Список пациентов, карточки, добавление записей |
| Врач | `doctor_petrova` | `1234` | Список пациентов, карточки, добавление записей |
| Администратор | `admin` | `admin` | Всё то же + управление врачами |

## 6. Что умеет система

### Для врача

- **Список пациентов** — просмотр всех пациентов с поиском по ФИО или номеру карты (например: `Сидоров` или `P001`)
- **Карточка пациента** — полная информация: ФИО, дата рождения, телефон, анамнез, лечащий врач
- **История** в карточке разделена на два блока:
  - **Камни** — локализация, размер в мм, тип (оксалатный/уратный/фосфатный), дата обнаружения
  - **Обследования и приёмы** — дата, тип, результат
- **Добавление пациента** — через пункт меню «Добавить пациента»
- **Добавление камня** — кнопка «Добавить камень» на карточке пациента
- **Добавление обследования** — кнопка «Добавить обследование»
- **Добавление приёма** — кнопка «Добавить приём» (сохраняется как обследование с типом «Приём врача»)

### Для администратора

Всё вышеперечисленное плюс:

- **Список врачей** — через пункт меню «Управление врачами»
- **Просмотр пациентов врача** — кнопка «Посмотреть пациентов» у каждого врача
- **Добавление нового врача** — кнопка «+ Добавить врача»

## 7. API-эндпоинты

Базовый URL: `http://127.0.0.1:8000`

| Метод | URL | Описание |
|---|---|---|
| `GET` | `/` | Проверка работоспособности сервера |
| `POST` | `/login` | Авторизация |
| `GET` | `/users` | Список всех пользователей (врачей и администраторов) |
| `POST` | `/users` | Создать нового врача |
| `GET` | `/users/{id}/patients` | Пациенты конкретного врача |
| `GET` | `/patients` | Список всех пациентов |
| `GET` | `/patients/{id}` | Карточка пациента (включает камни и обследования) |
| `POST` | `/patients` | Создать нового пациента |
| `GET` | `/stones` | Список всех камней |
| `POST` | `/stones` | Добавить камень пациенту |
| `GET` | `/examinations` | Список всех обследований |
| `POST` | `/examinations` | Добавить обследование пациенту |

### Примеры запросов

**Авторизация:**
```
POST /login
Content-Type: application/json

{
  "username": "doctor",
  "password": "1234"
}
```
Ответ `200 OK`:
```json
{
  "success": true,
  "role": "doctor",
  "name": "Иванов Иван Иванович",
  "id": 1
}
```
Ответ `401 Unauthorized` при неверных данных:
```json
{"detail": "Invalid username or password"}
```

**Добавить камень:**
```
POST /stones
Content-Type: application/json

{
  "patient_id": 1,
  "location": "Левая почка",
  "size_mm": 8.5,
  "stone_type": "оксалатный",
  "detected_date": "2024-06-01"
}
```

**Добавить обследование:**
```
POST /examinations
Content-Type: application/json

{
  "patient_id": 1,
  "date": "2024-06-08",
  "examination_type": "УЗИ почек",
  "result": "Камень 8 мм без обструкции",
  "doctor_id": 1
}
```

## 8. Частые ошибки и решения

### «Ошибка входа» / «Failed to fetch»

**Причина:** бэкенд не запущен или запущен на другом порте.  
**Решение:** убедитесь, что сервер работает на `http://127.0.0.1:8000` (см. шаг 3.2).

---

### Страница пустая после входа

**Причина:** браузер заблокировал запросы из `file://` (CORS).  
**Решение:** используйте Способ А из раздела 4 — запустите фронтенд через `python -m http.server`.

---

### `ModuleNotFoundError: No module named 'fastapi'`

**Причина:** зависимости не установлены.  
**Решение:**
```bash
pip install -r server/requirements.txt
```

---

### `Address already in use` (порт 8000 занят)

**Причина:** порт 8000 уже используется другим процессом.  
**Решение:** запустите на другом порте:
```bash
python -m uvicorn server.main:app --reload --port 8001
```
После этого нужно обновить `API_BASE_URL` в файле `client_interface/api.js`:
```js
const API_BASE_URL = 'http://127.0.0.1:8001';
```

---

### Данные пропали после перезапуска сервера

**Причина:** это ожидаемое поведение — данные хранятся в памяти (in-memory).  
При каждом запуске сервер загружает только тестовые данные, прописанные в `server/main.py`.  
Это ограничение текущего прототипа (см. раздел 9).

## 9. Известные ограничения

Система является учебным прототипом. Перед использованием в реальных условиях необходимо:

| Ограничение | Описание | Что нужно сделать |
|---|---|---|
| **Нет постоянного хранилища** | Данные хранятся только в памяти процесса и теряются при перезапуске | Подключить PostgreSQL (схема таблиц описана в `docs/documentation.md`) |
| **Нет хэширования паролей** | Пароли хранятся и сравниваются в открытом виде | Использовать `bcrypt` или `passlib` |
| **Нет JWT-авторизации** | После входа сервер не проверяет «кто делает запрос» — любой может обратиться к любому эндпоинту напрямую | Добавить JWT-токены и middleware-авторизацию в FastAPI |
| **Нет валидации на сервере** | Пустые поля и отрицательные значения проходят без ошибок | Добавить валидаторы Pydantic (например `@field_validator`) |
| **CORS открыт для всех** | `allow_origins=["*"]` — в продакшене это недопустимо | Ограничить до конкретного домена фронтенда |
| **Тестовые данные видны в коде** | Логины и пароли прямо в `server/main.py` | Перенести в переменные окружения или БД |
